# 収益計測セットアップ手順（Nori用）

このブログが「儲かっているか」を見えるようにするための設定です。
**パートAはもう完成（設定不要）。パートB（GA4）だけ、あなたの操作が必要です。**

---

## パートA：手動の収益ログ（✅ 完成済み・設定ゼロ）

A8・もしも・Amazon・楽天などの報酬は自動で取れないので、**週1回、数字を記録**します。

### やること
毎週、各ASPの管理画面で「今週の報酬額」を確認して、私（責任者）に伝えるだけ。
私が `data/revenue_log.json` に追記し、週次レポートにトレンドが出るようになります。

例：「今週はA8が500円、Amazonが120円、あとは0円だった」と言ってもらえれば記録します。

### 確認先（ブックマーク推奨）
- AdSense: https://adsense.google.com/adsense/
- A8.net: https://www.a8.net/
- もしも: https://af.moshimo.com/
- Amazon: https://affiliate.amazon.co.jp/
- 楽天: https://affiliate.rakuten.co.jp/
- アクセストレード: https://www.accesstrade.ne.jp/

## パートB：GA4（アクセス＆アフィリリンクのクリック計測）

「成約の前に、そもそもアフィリリンクが押されているか」を自動で見える化します。
コードはもう入っています。**あとは下の3ステップだけ。**

### ステップ1：GA4プロパティを作って測定IDを取る
1. https://analytics.google.com/ にアクセス（Googleアカウントでログイン）
2. 「管理」→「プロパティを作成」→ サイト名「Novlify」、URL `https://novlify.jp`
3. 作成後、「データストリーム」→ ウェブ → `https://novlify.jp` を登録
4. 表示される **測定ID（`G-XXXXXXXXXX` の形）** をコピー

### ステップ2：測定IDをサイトに設定する
Cloudflareの環境変数に登録します（パスワードではないので私が手順を案内できます）。
- 変数名：`PUBLIC_GA4_ID`
- 値：ステップ1でコピーした `G-XXXXXXXXXX`

→ この値を私に伝えてもらえれば、`wrangler.jsonc` への反映と再デプロイは私がやります。

### ステップ3（任意・後回しOK）：レポートの自動メール化
週次でPV・流入元を自動集計したい場合のみ。
1. Google Cloudで「サービスアカウント」を作成 → JSONキーをダウンロード
2. GA4の「管理」→「プロパティのアクセス管理」で、そのサービスアカウントを「閲覧者」で追加
3. GitHub Secrets に登録：
   - `GA4_PROPERTY_ID` = GA4プロパティID（数字のみ）
   - `GA4_SERVICE_ACCOUNT_JSON` = ダウンロードしたJSONの中身全部
   - ※AdSense収益も同じサービスアカウントで自動取得できます（`ADSENSE_PUBLISHER_ID` を追加）

## ⚠️ プライバシーの注意（GA4を有効にする前に）

GA4はアクセス解析のためにCookieを使います。日本では一般的ですが、
**プライバシーポリシーに「アクセス解析にGoogle Analyticsを利用しています」の一文**を入れておくのが安全です。
GA4を有効化するタイミングで、私が `privacy-policy` ページに追記します。
