## ① 未投稿ドラフトの確認

```bash
cat data/room_drafts/posted.json
ls data/room_drafts/*.md
```

`posted.json` の `posted` 配列に**含まれていない `.md` ファイル**が投稿対象です。  
未投稿が1件以上あれば、② → ③ → ④ の手順を即座に実行してください。

## ② 商品選択ルール（過去の学習から）

楽天市場で商品を検索したあと、以下の**優先順位**で1商品を選ぶこと。

### 必須条件（これを満たさない商品は選ばない）
| 条件 | 基準 |
|------|------|
| レビュー評価 | ★ **4.0 以上** |
| レビュー件数 | **20件以上**（件数が多いほど優先） |
| ROOMに投稿ボタン | 商品ページに存在すること |

### 優先条件（高いほど良い商品）
1. **送料無料** の商品を優先
2. **有名ブランド** を優先（後述のジャンル別ブランドリスト参照）
3. **検索キーワードが商品タイトルに含まれている**
4. **まとめ買い・セット商品** より単品商品を優先（コメントが書きやすい）
5. レビュー件数が **100件以上** あれば最優先

### ジャンル別 優先ブランドリスト

| ジャンル | 優先ブランド例 |
|---------|--------------|
| gourmet（食・健康） | 森永、明治ザバス、マイプロテイン、X-PLOSION、NICHIGA、大塚製薬 |
| gadget（ガジェット） | Anker、Logicool、エレコム、バッファロー、サンワサプライ、UGREEN |
| travel（旅行・アウトドア） | モンベル、コールマン、スノーピーク、ロゴス、キャプテンスタッグ |
| investment（投資） | ダイヤモンド社、日経BP、東洋経済など出版社商品（書籍・雑誌） |
| business（仕事効率） | コクヨ、プラス、キングジム、ブラザー、カシオ |

### 避けるべき商品
- レビュー件数が **5件未満**
- ショップ名が怪しい（中国語・英語のみ・記号だらけ）
- 商品画像が粗い・テキストだらけ
- 同一ページに複数の全く異なる商品が混在

## ③ 楽天ROOM投稿手順（Chrome MCP）

### 重要な技術的注意点（過去の学習から）
> ❌ `javascript_tool` は楽天市場商品ページでブロックされる（Cookie/query string data エラー）  
> ✅ 代わりに `find` + `read_page` でリンクのhrefを取得する

### 手順詳細

#### Step 1: 商品ページを開く
```
navigate( url="https://search.rakuten.co.jp/search/mall/{検索キーワード}/" )
```
- 検索結果から ② の条件に合う商品をクリック
- 商品ページが読み込まれるまで `wait(2秒)` してから次へ

#### Step 2: ROOMに投稿のURLを取得
```
find( query="ROOMに投稿", tabId=XXX )
→ ref_XXX が返ってくる
read_page( tabId=XXX, ref_id="ref_XXX" )
→ href="https://room.rakuten.co.jp/mix?itemcode=...&scid=we_room_upc60" が取得できる
```

#### Step 3: ROOMフォームを開く
```
navigate( url="https://room.rakuten.co.jp/mix?itemcode=...&scid=we_room_upc60", tabId=XXX )
wait(2秒)
```

#### Step 4: コメントを入力する
```
find( query="コメントテキストエリア", tabId=XXX )
→ ref_YYY が返ってくる

form_input( ref="ref_YYY", tabId=XXX, value="コメント本文\n\n#ハッシュタグ1 #ハッシュタグ2 ..." )
```

#### Step 5: 投稿する（完了ボタン）
```
find( query="完了ボタン", tabId=XXX )
→ ref_ZZZ が返ってくる（テキストが "完了" のボタン）

left_click( ref="ref_ZZZ", tabId=XXX )
wait(3秒)
screenshot()
```
→ 「コレ！完了!」画面が表示されれば成功 ✅

### コメントの書き方ガイド（過去の学習から）

| 項目 | ルール |
|------|--------|
| 文字数 | **160〜180文字**（500文字上限に対して余裕を持つ） |
| 行数 | **4〜5行**（改行で読みやすく） |
| 文体 | 一人称・体験談調（「〜してみたら」「〜だったんですが」） |
| 締め | `◎` で終わる（例：「〜が変わった気がします◎」） |
| ハッシュタグ | コメントの最後に空行を挟んで記載 |
| ハッシュタグ数 | **5〜7個**（`#楽天ROOM #楽天市場` は必ず含める） |

### 良いコメント例（gourmet_20260514 投稿1より）
```
ホエイ・ソイ・カゼインの違いを調べてみたら
「筋トレ後30分以内に飲むならホエイが一番」って知ってから
種類を変えました。
前は「なんとなく有名なやつ」を選んでたけど
目的に合わせて選ぶようになってから実感が変わった気がします◎

#楽天ROOM #楽天市場 #ホエイプロテイン #筋トレ #タンパク質 #運動後 #ガジェット
```

## ④ 投稿完了後に posted.json を更新

```bash
# posted.json の "posted" 配列に追加してコミット・プッシュ
git add data/room_drafts/posted.json
git commit -m "auto: mark room draft {ドラフト名} as posted"
git push
```

`posted.json` 更新時は `posted` 配列に `"{genre}_{date}"` 形式の文字列を追記する。

## generate.py 実行後のフロー

```
generate.py 実行 → data/room_drafts/{genre}_{date}.md 作成
  ↓
（次回Claudeセッション起動時）
① 未投稿ドラフトを自動検出
  ↓
② 各ドラフトの検索キーワードで楽天市場を検索 → 条件に合う商品を選択
  ↓
③ Chrome MCP で楽天ROOMに投稿（投稿1〜3件）
  ↓
④ posted.json を更新してプッシュ
```

## ⑤ 楽天ROOM公式・人気ブログから学んだ投稿知識

### ハッシュタグの重要ルール（公式より）

| ルール | 内容 |
|--------|------|
| `#オリジナル写真` | **ROOMで最もたくさん検索されているハッシュタグ**。オリジナル写真投稿時は必ず付ける |
| 半角 `#` を使う | 全角 `＃` やシャープ `♯` は検索リンクが機能しない → **必ず半角** |
| ピックアップ欄テーマタグ | 毎週金曜に更新されるテーマに沿ったタグを使うとピックアップ欄に掲載されやすい |

### 季節・イベント別おすすめハッシュタグ

| 時期 | 有効なタグ例 |
|------|------------|
| 1〜3月（新生活） | `#新生活準備` `#入園入学準備` `#春コーデ` `#バレンタイン` |
| 4〜6月（初夏） | `#母の日ギフト` `#梅雨対策` `#UV対策` `#2025春夏` |
| 7〜9月（夏〜秋） | `#暑さ対策` `#防災グッズ` `#秋先取り` `#スーパーSALE` |
| 10〜12月（冬末年） | `#クリスマスギフト` `#年末大掃除` `#2025冬` `#福袋` |
| 通年 | `#ふるさと納税` `#セールで買ったもの` `#新作コスメ` `#ペット用品` |

### コメントを読んでもらうための4つのテクニック

1. **具体的な使い方＋ベネフィット型**  
   例：「スマホと財布がちょうど入るミニバッグ。近所のカフェやお出かけに◎」
   
2. **感情を含める型**  
   例：「見た瞬間に『これだ！』ってなった。シンプルなのに高見えする◎」
   
3. **疑問形で惹きつける型**  
   例：「プロテインって何を選べばいいか迷いませんか？種類を変えてから実感が◎」
   
4. **自分の体験談型**（現在メインで使用中）  
   例：gourmet_20260514 投稿1のホエイプロテインコメント

### 投稿文字数について

| 情報源 | 推奨文字数 |
|--------|----------|
| 過去の実際の投稿経験（優先） | **160〜180文字** |
| 外部人気ブログ（参考） | 80〜150文字がベストとされる |

> 実績ある160〜180文字を基本とするが、「読みやすさ」が最優先。長くなりすぎず、短すぎず。

### 投稿のゴールデンタイム

- **夜9時以降**（夜のスマホチェック時間）
- 通勤時間（朝7〜9時）
- お昼休み（12〜13時）

> 自動化の場合は夜9時〜11時の間に投稿できると理想的。

### ピックアップ欄に掲載されるには

- **毎週金曜更新**のテーマタグを使う
- オリジナル写真 ＋ `#オリジナル写真` タグ
- 「紹介したい商品が正しく伝わる」コメント・写真にする
- 季節・イベントに合ったタグを複数付ける

### その他の注意点

- ハッシュタグは投稿ごとに**季節感のあるものを1〜2個差し替える**と新鮮さが出る
- 商品紹介コメントは「何の商品か正確に伝わること」が最重要（関係のない写真と組み合わせない）
- デメリットも正直に書くと信頼性が向上するが、ROOMでは基本的にポジティブな体験談調で◎

## ⑥ A8ネット・もしもアフィリエイト公式から学んだ記事作成・収益化知識

> このセクションは **ブログ記事を生成・改善する際に参照**する知識です（generate.py の記事品質向上に活用）。

---

### 【A8キャンパス公式】記事作成7ステップ

#### STEP1: 広告プログラム選定
- サイトのテーマとマッチした商品を選ぶ
- **実店舗で容易に買えるものは避ける** → ネット限定・ニッチな商品が成果に繋がりやすい
- 「ネットで売れるものか？」を必ず確認する

#### STEP2: キーワード選定
| 優先順位 | キーワード例 | 特徴 |
|---------|------------|------|
| 高 | `商品名 口コミ` `商品名 メリット デメリット` `商品名 比較` | **購買意図が高い** → 成果に直結 |
| 中 | `商品名 使い方` `商品名 おすすめ` | 比較検討段階 |
| 低 | `○○とは` `○○店舗` | 情報収集段階 → 成果には繋がりにくい |

- **ロングテール（3語以上）を狙う** → 競合が少なく上位表示しやすい
- ラッコキーワード・Googleキーワードプランナーで検索需要を必ず確認
- キーワードの詰め込みはGoogleペナルティ対象 → 自然な文章が優先

#### STEP3: 記事タイトルの書き方
- **キーワードを冒頭に配置**（検索結果で目立つ）
- **数字を含める**（例：「5選」「30分で」）
- **文字数は30〜40文字以内**（検索結果での省略を防ぐ）
- 記事執筆後にタイトルを見直すと改善しやすい

#### STEP4: 導入文の型
```
① 「こんな悩みありませんか？」→ 共感を誘う問いかけ
② 解決策を提示
③ 記事の概要を1〜2文で
```

#### STEP5〜6: 本文の構成
- 小見出し（H2/H3）を **3〜5個** に整理
- 各見出しで扱うべき視点：使い心地・競合比較・おすすめ理由・ターゲット明確化
- **太字・マーカーで要点を強調**
- 画像は最低3種類（全体像・特徴・細部）
- **主張には客観的な根拠を付ける**（統計データ・行政機関情報など）
- 個人の感想のみでは根拠として不十分

#### STEP7: 記事公開前の確認
- **季節商材は需要ピークの3ヶ月前に公開**（検索エンジンに認識される時間が必要）
- アイキャッチ画像を設定
- SNS共有は「自分の言葉で丁寧に」（自動投稿より個人的な発信が効果的）

---

### 【A8キャンパス公式】SEO対策の基本3要素

#### 1. 内部SEO
- HTMLタグを正しく階層化（h1→h2→h3）
- 画像の `alt` 属性を設定
- XMLサイトマップを送信

#### 2. コンテンツSEO（最重要）
- **検索エンジンではなく、ユーザーのための情報発信**を徹底
- **E-E-A-T** を意識する

| 要素 | 内容 |
|------|------|
| 専門性（Expertise） | そのジャンルの専門知識・実体験があること |
| 権威性（Authoritativeness） | そのジャンルの専門家として認識されること |
| 信頼性（Trustworthiness） | 事実に基づく情報・運営者情報の明記 |

- **YMYL領域**（金融・健康・法律）は E-E-A-T がさらに厳格に評価される → 要注意

#### 3. 外部SEO
- 良質なコンテンツで自然な被リンクを獲得
- **リンク購入・サテライトサイトはペナルティ対象** → 絶対NG

### 【A8キャンパス公式】ジャンル選定の基準

稼げるジャンルの3条件：

1. **自分が興味・経験を持っている**（専門家レベル不要、実体験で十分）
2. **市場が一定以上の大きさ**（推奨ジャンル：仕事・金銭・住居・恋愛・美容健康・グルメ）
3. **継続的な需要がある**（流行廃りのないテーマ）

### 【もしもアフィリエイト公式】楽天・Amazon報酬率の比較

楽天市場を紹介する際の報酬率（1万円の商品を売った場合の例）：

| ASP | 報酬率 | 1万円商品の報酬額 |
|-----|--------|----------------|
| **もしもアフィリエイト** | 3% ＋ W報酬10%上乗せ | **約330円** ← 最もお得 |
| 楽天アフィリエイト（純正） | 3% | 300円 |
| A8ネット | 2.2% | 220円 |

> ✅ **楽天商品のアフィリエイトは「もしもアフィリエイト」経由が最もお得**

Amazon商品の報酬率：
- もしもアフィリエイト ≧ A8ネット（カテゴリにより差あり）
- 2024年8月から Amazonの紹介料上限が廃止（以前は1,000円/商品上限）
- 楽天は上限制限が継続中

### 【もしもアフィリエイト公式】かんたんリンク機能

- Amazon・楽天・Yahoo!ショッピングを **1つのリンクに同時掲載** できる
- ユーザーが好みのプラットフォームから購入できるため **CVR（成約率）が上がる**
- WordPressへのHTMLコード貼り付けで簡単に使える
- プラグイン「ポチップ」と組み合わせるとさらに高度なカスタマイズが可能

---

### 【もしもアフィリエイト公式】登録・Amazon審査の条件

| 条件 | 詳細 |
|------|------|
| もしもアフィリエイト登録 | **5記事以上**が必要 |
| Amazon広告の利用 | **10記事以上**が必要 |
| Amazon審査 | 本家より通りやすい（3件成約条件なし） |
| 免責表記 | 「Amazonのアソシエイトとして〜収入を得ています」の記載が必須 |

> 本家Amazonアソシエイトは審査が厳しい（半年で3件成約が必要）が、もしもアフィリエイト経由は条件が緩い。
